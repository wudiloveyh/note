## 漏洞探测

### 默认路径探测-未授权

如果配置不当的话不需要druid密码就可以访问druid

```diff
/druid/index.html
```

### 默认路径探测-druid

若依的默认druid路径：

```diff
/druid/login.html
```

直接拼接，如果成功访问的话就存在druid后台

### 默认路径探测-默认api

有时候存在默认的api，druid就在这个api下面

```diff
/prod-api/druid/login.html
/dev-api/druid/login.html
```

拼接成功的话就存在druid后台

![image-20251116102527146](https://cdn.jsdelivr.net/gh/wudiloveyh/my-image-bucket@main/images/20251116102527189.png)

### 路径探测-开发自定义的api

有的时候会把默认的api改个名，你得试错，在登录界面登录错误一次，然后在历史包会出现对应的api，然后去拼接就可以了

那么拼接的地址就是：

```diff
/{发现的api}/druid/login.html
```

### 默认路径探测-总结

常见的路径：

```diff
/druid/index.html
/druid/login.html
/prod-api/druid/login.html
/prod-api/druid/index.html
/dev-api/druid/login.html
/dev-api/druid/index.html
/api/druid/login.html
/api/druid/index.html
/admin/druid/login.html
/admin-api/druid/login.html
```

其他在实战中补充就好了
